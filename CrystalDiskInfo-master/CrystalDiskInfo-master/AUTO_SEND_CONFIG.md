# Автоматическая отправка данных при запуске

## Описание

Добавлена возможность автоматической отправки данных на сервер при запуске программы. Отправка происходит в фоновом режиме без показа диалогов и сообщений.

## Настройка

Добавьте в файл конфигурации `DiskInfo.ini` в секцию `[Server]`:

```ini
[Server]
AutoSendOnStartup=1
```

Где:
- `AutoSendOnStartup=1` - автоматическая отправка включена
- `AutoSendOnStartup=0` - автоматическая отправка выключена (по умолчанию)

## Поведение

- При `AutoSendOnStartup=1`: При запуске программы, после завершения сбора данных о дисках, данные автоматически отправляются на сервер по умолчанию
- Отправка происходит в фоновом режиме без показа диалогов подтверждения и сообщений об ошибках
- Если сервер по умолчанию недоступен, автоматически пробуются другие настроенные серверы
- При ошибках отправки пользователю не показываются сообщения (все происходит тихо)
- Все действия логируются через `DebugPrint` для отладки

## Пример конфигурации

```ini
[Server]
AutoSendOnStartup=1

[Server_1]
Name=Backup Server
Host=192.168.100.106
Port=5000
Default=1

[Server_2]
Name=Main Server
Host=10.3.3.7
Port=5000
Default=0
```

**Важно:**
- Секции серверов должны называться `[Server_1]`, `[Server_2]`, `[Server_3]` и т.д. (с подчеркиванием)
- Только один сервер должен иметь `Default=1` (сервер по умолчанию)
- Остальные серверы должны иметь `Default=0`
- Флаг `AutoSendOnStartup=1` добавляется в секцию `[Server]`

## Примечания

- Автоматическая отправка происходит только если в системе обнаружены диски
- Если ни один сервер не доступен, отправка просто завершается без ошибок (тихо)
- Данные отправляются на endpoint `/api/hdd_collect/v2`
